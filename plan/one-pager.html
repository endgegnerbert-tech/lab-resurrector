Problem

The Education Gap

  • 9 out of 10 schools in developing countries have no physics lab
  • A real school lab costs over $5,000 — out of reach for most
  • Existing digital solutions (PhET, YouTube) are passive: students watch, they don't experiment

The Opportunity

  • 85% of students worldwide own a smartphone
  • Every phone already has a browser, a screen, and internet access
  • Gen Z prefers interactive learning over passive video
  • "The phone is the lab" — not a substitute, a better tool
Solution

flabs is a browser-based physics lab with an AI assistant that controls simulations live. The key innovation: the AI runs in your browser. Your API key never touches the server.

How It Works

  1. Student types a question or hypothesis in the chat
  2. Browser-side AI agent calls the LLM API directly using the student's own key
  3. Agent controls the simulation (change params, reset, switch scenes) via tool calls
  4. Student sees live changes, measurements, and AI explanation simultaneously

Why Different

  • Server has zero AI dependencies — no SDK, no WebSocket, no secrets
  • Zero vendor lock-in — 18+ providers, user chooses
  • Zero signup — open browser, enter key if you want AI
  • Zero external frontend code — pure Canvas 2D, no React/Vue/jQuery
Product

Water Rocket Lab

Realistic 2-bottle rocket with pressure thrust, Tsiolkovsky rocket equation, and projectile motion. Adjust pressure (1-6 bar), water ratio (10-80%), and launch angle (10-80 deg) in real-time. Live: flight time, max height, range, velocity.

Wave Interference Lab

Two coherent point sources with visible superposition pattern. Adjust wavelength, source distance, phase shift, and amplitude. Live: max deflection, relative intensity, node count.

AI Tools

  • sim_set_param — change anything live
  • sim_reset — back to defaults
  • sim_switch_scene — switch experiments
  • sim_highlight — focus on elements

Data & Export

  • Live measurement streaming (every 8 frames)
  • Formula panel with computed predictions
  • CSV + JSON export of all measurements

Space System

  • Isolated iframe sandbox per experiment
  • postMessage bridge for controls
  • Create new labs on demand
Architecture
Browser (AI Agent) — js/ai/agent.js stores your key in localStorage, calls LLM API directly via fetch()
    |— sim_set_param, sim_reset, sim_switch_scene, sim_highlight
    |— Canvas 2D physics rendering
    |— Chat UI + live data panel
    |— CSV/JSON export
| HTTP (static files + REST API)
Server (Express) — no AI, no WebSocket, no secrets
    |— Serves index.html, experiments/, sources/
    |— REST: GET/POST/DELETE /api/spaces
    |— Health: GET /api/health

Security

Zero server secrets — this is an architectural fact, not a marketing claim. The server has no AI dependencies, no WebSocket, no API key storage. The AI agent runs in the browser. Keys are stored in localStorage and sent directly to LLM APIs via fetch(). The server never receives, stores, or transmits API keys. Full documentation: SECURITY.md.
