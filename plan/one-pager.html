How It Works

No phases. No templates. The agent builds experiments directly.

  1. Student requests an experiment via the AI chat
  2. Agent calls physics_formula_lookup + physics_model_plan against a local library of 30+ verified physics formulas (sources: myphysicslab, OSP, matter-js)
  3. Agent calls emet if local knowledge is insufficient — fetches cited, authoritative physics from live sources (arXiv, papers, educational databases)
  4. Agent writes four files into the experiment space: experiment.json (parameters+measurements), index.html (sandboxed iframe), sketch.js (Canvas 2D physics engine + measurement emission), sources.json (license-safe attribution)
  5. Agent calls space_verify_current — validates all files before declaring success
  6. Experiment runs live in the browser. Student adjusts parameters, watches measurements, asks follow-ups.

Pipeline: request → grounded lookup → write → verify → live in seconds

What Makes This Different

No Phased Flow

The agent does not ask "what phase do you want". It builds the complete experiment, validates it, and presents it. The student starts interacting immediately.

Built-in Physics Grounding

Every formula has validity limits, units, and source IDs. The agent never invents physics. When local knowledge is insufficient, emet provides cited research.

Bring Your Own API Key

Zero server-side secrets. Each browser creates an ephemeral pi agent session with the user's own key. No signup, no shared accounts, no vendor lock-in.

Agent Builds Experiments From Scratch

Not a template picker. The agent writes real Canvas 2D simulation code with correct physics. Every experiment is a fresh, isolated iframe sandbox.

Architecture

       | WebSocket (JSON streaming)
Node.js Server (Express + ws + REST API)
       | pi SDK AgentSession
12 Custom Physics Tools + emet + pi built-in tools
       | User's AI Model (DeepSeek, Claude, Gemini, Ollama...)

Key property: server holds zero secrets. API keys set via authStorage.setRuntimeApiKey() per browser session — ephemeral, in-memory, never written to disk.
