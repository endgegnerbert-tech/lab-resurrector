# 13 — Fragen für den nächsten Build-Schritt

Diese Fragen blockieren den Plan nicht, aber helfen beim nächsten Implementationsschnitt.

## Source Library

1. Sollen wir Repos vollständig unter `sources/repos/` ablegen oder nur Summaries/Templates extrahieren?
2. Sollen große Repos ausgeschlossen werden und nur relevante Dateien gespeichert werden?
3. Sollen Libraries lokal vendored werden statt CDN?

## Spaces

4. Soll jeder Space eine Version bekommen?
5. Soll es Duplizieren/Forken von Spaces geben?
6. Soll ein Space exportierbar sein als einzelner HTML-Ordner?

## UI

7. Sollen Daten/Formel unten gleichzeitig sichtbar sein oder als Tabs?
8. Soll Chat rechts immer offen sein oder einklappbar?
9. Soll es eine Teacher-Ansicht später geben?

## Agent

10. Soll der Agent automatisch neue Spaces bauen, wenn er sicher ist, oder immer kurz ankündigen?
11. Soll der Agent nach jedem Build eine kurze Hypothesenfrage stellen?
12. Soll der Agent schlechte/zu komplexe Wünsche automatisch in ein vereinfachtes Modell übersetzen?

## MVP

13. Was ist der erste End-to-End Demo-Satz?
    - Vorschlag: „Ich will verstehen, warum die Masse beim Pendel egal ist.“
14. Was ist der zweite Demo-Satz aus einem anderen Bereich?
    - Vorschlag: „Mach mir ein Experiment zu Interferenz von Wellen.“
